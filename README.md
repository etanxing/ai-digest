# AI Digest — `ai-digest.isawesome.work`

The AI-curated AI briefing. 20+ sources. 3 stories. 5 minutes. Published daily.

## Architecture

```
GitHub repo (this)
├── web/          Next.js static export — EN + ZH blog
├── worker/       Cloudflare Worker — daily pipeline (cron 09:00 UTC)
├── shared/       sources.json, TypeScript types
└── content/      Markdown posts (auto-generated by worker)
```

**Flow:** Worker fetches RSS → scores → Workers AI curates & writes → Workers AI translates → commits markdown → CF Pages rebuilds site.

## Local development

```bash
npm install
npm run dev          # site at http://localhost:3000/en
```

## Build

```bash
npm run build        # outputs static HTML to out/
```

## Go-live checklist

### 1. Create GitHub repo + push

```bash
gh repo create ai-digest --public --source=. --remote=origin --push
```

### 2. Set `GITHUB_REPO` in worker

Edit `worker/wrangler.toml`:
```toml
[vars]
GITHUB_REPO = "your-github-username/ai-digest"
```

### 3. Create CF Pages project

- Dashboard → Pages → Create project → Connect to GitHub → select `ai-digest`
- Build command: `npm ci && npm run build`
- Build output: `out`
- Deploy

### 4. Add custom domain

CF Pages → Custom domains → Add `ai-digest.isawesome.work`
(CNAME is auto-created since `isawesome.work` is already in CF)

### 5. Create KV namespaces + deploy worker

```bash
cd worker

# Create KV namespaces (copy the IDs into wrangler.toml)
wrangler kv namespace create SEEN_ITEMS
wrangler kv namespace create SOURCE_STATE

# Edit wrangler.toml: paste the real KV IDs

# Set GitHub token (fine-grained PAT, Contents: read+write on this repo)
wrangler secret put GITHUB_TOKEN

# Deploy
wrangler deploy
```

### 6. Test the pipeline (dry run)

```bash
# In wrangler.toml, set DRY_RUN = "true"
wrangler dev
curl "http://localhost:8787/__scheduled"
# Check console output — should print digest preview without committing

# Then set DRY_RUN = "false" and re-deploy for real
```

## Content

## Sources
